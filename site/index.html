› get it running

one-liner

Detects your OS + architecture, drops the binary in /usr/local/bin (or ~/.local/bin).

curl -fsSL https://raw.githubusercontent.com/aleandros/mdx/main/install.sh | sh

cargo

Published as mermd on crates.io (the mdx name was taken).

cargo install mermd

from source

Rust toolchain required. Binary lands in target/release/mdx.

git clone https://github.com/aleandros/mdx.git
cd mdx
cargo build --release

then use it

# render a file
mdx README.md

# pipe anything
cat CHANGELOG.md | mdx

# watch mode
mdx -W notes.md

# embed in another TUI
mdx embed docs/architecture.md

› why

Markdown is everpresent in day-to-day workflow, and glow spoils you — until the doc hits a Mermaid block and you're back to squinting at raw source or alt-tabbing to a browser.

mdx is the fix. Same "just read this thing" ergonomics, but diagrams render where you already are.

Diagram support is scoped. Today mdx renders graph / flowchart and sequenceDiagram. Class, state, ER, gantt, and others are not yet supported — PRs welcome.

stop alt-tabbing.

Render the whole thing where you're already working.
